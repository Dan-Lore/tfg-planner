# Открытые вопросы

При ответе — переносить решение в [specification.md](specification.md) / [architecture.md](architecture.md) и обновлять статус.

## Решено

| # | Вопрос | Решение | Дата |
|---|--------|---------|------|
| Q1 | Откуда брать рецепты? | **Парсер** репозитория [Modpack-Modern](https://github.com/TerraFirmaGreg-Team/Modpack-Modern) → pack data. См. [parser.md](parser.md) | 2026-06-17 |
| Q4 | Направление расчёта? | **Двусторонний** по продуктам; вся схема пересчитывается. Машины настраивает пользователь | 2026-06-17 |
| Q5 | Энергия EU/t? | **Да, поэтапно.** Без заглушек — только реальные данные или скрытый UI. Kanban K-003 | 2026-06-17 |
| Q8 | Язык UI? | **Русский + английский** | 2026-06-17 |
| Q10 | Публикация по ссылке? | **Только в планах** (v2/v3). MVP = `.tfgp`. См. [roadmap.md](roadmap.md) | 2026-06-17 |
| Q16 | Масштабирование: дублировать узлы или умножать потоки? | **Три режима:** A — ручное копирование линии; B — «Умножить выходы» выделенных машин; C — целевой продукт на выходе. См. spec §3.5 | 2026-06-17 |
| Q7 | Дробные множители (×1.5)? | **Да.** Умножение/деление и целевые скорости могут быть нецелыми. `machineCount` после пересчёта — **целое** (§3.5.1) | 2026-06-17 |
| Q17 | Округление `machineCount`? | **Вверх (ceil)** + полный пересчёт схемы. Единственная стратегия | 2026-06-17 |
